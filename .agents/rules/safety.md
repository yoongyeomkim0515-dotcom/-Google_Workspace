# Safety Rules

> 이 파일은 공통 안전 규칙 라이브러리다. 각 도구의 글로벌 룰이 우선하며, 이 파일은 세부 참조용으로 사용한다.

## 1. 절대 금지
- `.env`, `.pem`, `.key`, `.p12`, `.pfx`, private key, API token, secret, credential 파일을 읽거나 수정하지 않는다.
- 비밀키, 토큰, 개인정보, 인증 쿠키, 세션 값을 코드·로그·응답에 출력하지 않는다.
- 사용자가 제공하지 않은 민감정보를 추측하거나 복원하려고 시도하지 않는다.
- 보안 경고를 우회하거나 숨기지 않는다.

## 2. 사용자 승인 필요
아래 작업은 사용자 승인 없이 실행하지 않는다.

- 파일/폴더 삭제, 대량 덮어쓰기, 히스토리 삭제
- 배포, 릴리즈, 운영 서버 변경
- 운영 DB 쓰기, 삭제, 마이그레이션
- 결제, 환불, 정산, 메일 대량 발송
- 계정 설정, 권한, 토큰, 인증 정보 변경
- 외부 패키지 설치, 시스템 설정 변경

## 3. 안전한 실행 순서
- 외부 API, DB, 결제, 메일, 배포 작업은 mock, dry-run, sandbox를 먼저 사용한다.
- 위험한 작업 전에는 영향 범위, 되돌리기 방법, 필요한 승인을 설명한다.
- 사용자가 승인해도 비밀키를 직접 요청하거나 출력하지 않는다.
- 승인 기록이 모호하면 실행하지 않고 다시 확인한다.

## 4. 실패 처리
- 동일 오류가 3회 반복되면 자체 시도를 중단한다.
- 중단 후에는 로그, 근본 원인 후보, 우회로, 사용자가 선택할 수 있는 다음 단계를 보고한다.
- 실행하지 않은 검증을 실행했다고 말하지 않는다.
