# Harness Rules

> 이 파일은 AI 코딩 에이전트의 작업 절차를 표준화하는 공통 하네스(Harness) 규칙이다.

## 1. 작업 시작 전
- 사용자의 목표를 한 문장으로 요약한다.
- 관련 파일과 폴더 구조를 먼저 확인한다.
- 기존 코드 스타일, 의존성, 테스트 명령어, 빌드 명령어를 확인한다.
- 모르는 것은 추측하지 말고 확인 가능한 정보와 불확실한 정보를 분리한다.

## 2. 파일 수정 전
- 수정 대상 파일을 먼저 읽는다.
- 한 번에 너무 큰 변경을 하지 말고 작은 단위(Small Chunk)로 나눈다.
- 기존 공개 API, 파일 구조, 주석, 문서를 사용자 요청 없이 임의로 삭제하지 않는다.
- 자동 생성 파일이나 잠금 파일은 필요한 경우에만 수정한다.

## 3. 터미널 명령
- 터미널 명령 실행 전 목적을 한 줄로 설명한다.
- 위험하거나 되돌리기 어려운 명령은 사용자 승인 전 실행하지 않는다.
- 설치 명령은 필요한 이유를 설명하고 승인 후 실행한다.
- 명령 실패 시 에러 메시지를 근거로 원인을 분석한다.

## 4. 검증
- 수정 후 가능한 테스트, 빌드, 린트, 실행 확인을 수행한다.
- 테스트를 실행할 수 없으면 이유와 수동 검증 방법을 제시한다.
- 결과 확인은 사용자가 따라 할 수 있게 명령어, 화면 확인 포인트, 예상 결과를 제시한다.

## 5. 기록
- 중요한 결정, 제약, 실패 원인, 다음 작업은 프로젝트 메모리 파일에 기록하는 것을 권장한다.
