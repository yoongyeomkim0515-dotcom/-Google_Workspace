# Frontend Polish Skill

## Goal
프론트엔드 화면을 작동만 하는 수준에서 사용자가 보기 좋고 이해하기 쉬운 완성도 있는 UI로 개선한다.

## When to Use
- UI가 밋밋하거나 정리가 필요할 때
- 반응형 레이아웃이 필요한 때
- 접근성(Accessibility), 시맨틱 HTML(Semantic HTML), SEO를 보강할 때
- Empty State, Loading State, Error State가 필요한 때
- 다크모드, 애니메이션, 카드 레이아웃, 시각적 위계를 다듬을 때

## Steps
1. 현재 컴포넌트 구조와 스타일 방식을 확인한다.
2. 화면의 목적과 핵심 사용자 행동을 파악한다.
3. 레이아웃, 간격, 타이포그래피, 색 대비, 시각적 위계를 점검한다.
4. Placeholder만 남기지 말고 실제 예시 콘텐츠, 빈 상태, 로딩 상태, 에러 상태를 설계한다.
5. 시맨틱 HTML과 접근성 속성을 확인한다.
6. 필요한 경우 title, description, Open Graph 등 기본 SEO 메타 태그를 제안한다.
7. 애니메이션과 시각 효과는 성능과 접근성을 해치지 않는 범위에서만 적용한다.
8. 변경 후 브라우저에서 확인할 수 있는 검증 포인트를 제시한다.

## Constraints
- 미학적 개선이 기능 안정성, 접근성, 성능보다 우선하지 않는다.
- 과도한 애니메이션이나 불필요한 의존성 추가를 피한다.
- 새 패키지 설치가 필요하면 사용자 승인을 요청한다.
- 기존 디자인 시스템이 있으면 우선 따른다.

## Output
- 개선 요약
- 수정한 컴포넌트/파일
- 브라우저 확인 포인트
- 접근성/반응형 체크 포인트
