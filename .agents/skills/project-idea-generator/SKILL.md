# Project Idea Generator Skill

## Goal

사용자의 맥락(기술 수준, 도메인 관심사, 목적)을 반영하여 실행 가능한 프로젝트 아이디어 3개를 구체적으로 제시한다.

## When to Use

- 새로운 사이드 프로젝트 주제를 찾을 때
- 포트폴리오에 추가할 프로젝트를 고민할 때
- 특정 기술 스택을 연습할 프로젝트가 필요할 때
- 창업 아이디어나 비즈니스 프로젝트 주제를 탐색할 때
- 브레인스토밍이나 기획 초기 단계에서 방향을 잡을 때

## Steps

1. 사용자 맥락을 파악한다.
   - 보유 기술 스택 또는 연습하고 싶은 기술이 있는지 확인한다.
   - 관심 도메인(비즈니스, 개발 툴, 생활 편의, 교육, 데이터 등)을 파악한다.
   - 프로젝트 목적(포트폴리오, 수익화, 학습, 개인 사용)을 확인한다.
   - 맥락 정보가 없으면 다양한 도메인에서 균형 있게 선정한다.

2. 아이디어 3개를 서로 다른 도메인 또는 접근 방식으로 선정한다.
   - 각 아이디어는 실현 가능한 범위여야 한다 (1인 개발 기준).
   - 3개 아이디어가 서로 중복되지 않도록 한다.
   - 사용자 맥락이 있으면 그에 맞게 필터링한다.

3. 각 아이디어를 아래 형식으로 출력한다.

## Output

아이디어마다 아래 구조로 출력한다:

```
### 💡 [프로젝트명]

**한 줄 설명:** [무엇을 만드는지 한 문장으로]

**핵심 기능:**
- [기능 1]
- [기능 2]
- [기능 3]

**예상 난이도:** 초급 / 중급 / 고급
**예상 기간:** [예: 1~2주 / 1개월]
**추천 기술 스택:** [예: Python, FastAPI, SQLite]
**왜 만들면 좋은가:** [학습 가치 또는 실용 가치 한 줄]
```

3개 아이디어 출력 후, 아래를 덧붙인다:
- 더 자세히 알고 싶은 아이디어가 있으면 말해달라는 안내
- 도메인이나 기술 스택 조건을 주면 더 맞춤화된 아이디어를 제시할 수 있다는 안내

## Constraints

- 실현 불가능하거나 지나치게 거대한 아이디어(예: SNS 플랫폼 전체 구축)는 제안하지 않는다.
- 아이디어 3개는 반드시 서로 다른 성격(예: 개발 툴 / 비즈니스 / 생활 편의)으로 구성한다.
- 사용자가 특정 기술 스택을 언급한 경우 해당 스택 중심으로 제안한다.
- 수익화 목적이 있는 경우 예상 비즈니스 모델도 한 줄 추가한다.
- 아이디어는 매 요청마다 새로 생성하며 이전과 동일한 내용을 반복하지 않는다.
